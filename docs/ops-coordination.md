# Coordination Notes

Maintainers may track larger changes in an external issue tracker or Ops
catalogue. Keep those records concise enough that a new contributor can pick up
the work without private chat context.

For substantial changes, use a parent item plus focused child tasks when the
work spans independent areas such as documentation, runtime behavior, security
review, and release validation.

Each tracked item should include:

- Rationale: why the work matters now.
- Objective: what success looks like.
- Scope: what is included and excluded.
- Implementation guidance: how to execute safely.
- References: files, docs, issues, and related tasks.

Keep repo-specific operational guidance in [../README.md](../README.md),
[RUNBOOK.md](RUNBOOK.md), and [SECURITY_MODEL.md](SECURITY_MODEL.md).
